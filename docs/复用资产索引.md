# 复用资产索引

本文档记录本项目中可以被 AI 直接复用的结构、代码模式、命令和验证方法。AI 新增模块、页面、接口、脚本时，应先搜索本文档。

## 记录规则

新增复用资产时 MUST 使用以下结构：

```md
## 资产：一句话名称

适用场景：
- 什么时候使用

来源：
- `path/to/file`

复用方式：
- 怎么复制或参考

替换点：
- 需要替换哪些变量、字段、路径

注意事项：
- 容易出错的点
```

## 资产：标准后台列表页结构

适用场景：

- 新增业务模块列表页
- 调整已有模块列表页

来源：

- `frontend/src/views/workOrder/list.vue`

复用方式：

- 复用筛选区、表格区、分页区布局
- 复用后端分页和排序参数
- 按模块替换 API、筛选字段、表格列

替换点：

- API 方法
- `filters`
- `allFilters`
- 表格列
- 新增、编辑、详情、删除路由

注意事项：

- 第一行筛选区不要换行
- 新增按钮保持靠右
- 分页区不要被表格挤压
- 排序必须传给后端处理

## 资产：标准表单页结构

适用场景：

- 新增模块的新增、编辑页面

来源：

- `frontend/src/views/workOrder/form.vue`

复用方式：

- 复用三列 grid
- 复用底部取消和保存按钮
- 复用新增、编辑模式判断
- 复用表单校验和保存流程

替换点：

- 表单字段
- 校验规则
- 详情加载 API
- 新增 API
- 更新 API
- 返回路径

注意事项：

- keep-alive 激活时新增页要重置
- 编辑不同 id 时要重新加载
- 保存前必须 `validate`

## 资产：标准详情页结构

适用场景：

- 新增模块详情页

来源：

- `frontend/src/views/workOrder/detail.vue`

复用方式：

- 复用基本信息、历史记录、单据信息结构
- 复用空值显示规则
- 复用底部按钮

替换点：

- 详情 API
- 字段展示
- badge 映射
- 历史记录 API

注意事项：

- 单据信息固定在底部
- 空值显示 `-`
- 历史记录字段中文名只用于展示

## 资产：服务启动和验证命令

适用场景：

- 修改代码后重启服务
- 检查服务是否正常

来源：

- `AGENTS.md`

复用方式：

- 后端使用 `project-template-backend` screen 会话
- 前端使用 `project-template-frontend` screen 会话
- 后端端口 `3101`
- 前端端口 `3102`

替换点：

- 复制到新项目实例时替换项目路径、端口、日志文件、screen 名称

注意事项：

- 不要占用 `3001`
- 不要占用 `3002`
- 修改代码后必须按需验证
