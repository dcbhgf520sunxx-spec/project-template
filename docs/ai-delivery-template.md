# AI 交付单模板

每次 AI 完成业务变更后，按以下格式向用户交付：

## 变更内容

- 页面/组件：
- 接口/权限/菜单：
- 数据库：无 / 待确认表结构设计：

## 自动门禁

- 路由菜单权限检查：
- 前端组件审查与构建：
- 后端测试：
- migration 状态：
- 浏览器冒烟：页面、操作步骤、结果：
- 页面语义：操作列文字动作与“更多”规则、标题状态标签、状态操作位置、删除/状态弹窗组件：
- 标签色彩：普通分类标签是否使用 `AdminTag` 默认样式，且未自行指定颜色：
- 列表列行为：左右固定列、列宽拖拽、横向滚动、列头排序、创建人和创建时间位于操作列之前：
- 接口运行时契约：读取结果、写入业务对象结果：
- 状态流转契约：允许/禁止流转、附加字段、前后端一致性：
- 菜单授权：完整地址（含查询参数）、子菜单独立授权、一级菜单顺序：

## 验收

- 验收环境和地址：
- 建议验证的业务动作：
- 已知风险或未验证项：

## 下一步

- 无表结构变化：请完成最终业务验收；确认发布后由 AI 部署。
- 有表结构变化：请先确认表结构设计；AI 随后执行 migration 并提供验收环境。
